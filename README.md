# Dob

## The Open Version Control System Specification

Dob is an open specification for designing and implementing modern Version Control Systems.

Dob does not provide one official implementation. Instead, it defines the rules, formats, architecture, and behaviors that independent developers and organizations can use to build their own Dob-compatible systems.

---

# 🌐 Philosophy

Dob follows the same philosophy as the **Doublers project**:

* Open specifications
* Independent implementations
* Clear technical documentation
* Freedom to experiment
* Community-driven development

Similar to how multiple processors can implement an ISA specification, multiple Version Control Systems can implement Dob Spec.

---

# 📜 Dob Spec

**Dob Spec** is the official specification that defines the Dob standard.

Dob Spec describes:

* Repository formats
* Data structures
* Version tracking methods
* Commit models
* Branching rules
* Synchronization behavior
* Compatibility requirements

Any implementation following Dob Spec can be considered Dob-compatible.

---

# 🧠 Architecture Model

Dob Spec defines a common foundation:

```
+--------------------------------+
|    Dob-Compatible Systems      |
|  (Independent Implementations) |
+--------------------------------+
|          Dob Spec              |
|   Official Specification       |
+--------------------------------+
|     Storage & Data Rules       |
|     Defined by Standard        |
+--------------------------------+
```

Different implementations may have:

```
Implementation A
Implementation B
Implementation C
```

while still following the same Dob Spec.

# 🛠 Implementation Freedom

Developers are free to create:

* Command-line tools
* Graphical clients
* Distributed systems
* Enterprise implementations

# 🔗 Relationship With Doublers

Dob uses the same idea as Doublers:

**Dob:**
A version control specification that allows different software implementations.

The goal is not one program.

The goal is a standard that others can build upon.
